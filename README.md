# MURU-BENCH

**Mathematical Reasoning Under Uncertainty Benchmark** — 3,000 problems for evaluating whether language models can reason correctly when the answer is genuinely uncertain.

> Existing math benchmarks (GSM8K, MATH, MMLU) score a single number. **MURU-BENCH instead requires every prediction to carry four outputs**: a point estimate, a confidence interval, a stated confidence probability, and a named reasoning framework. A confident guesser and a calibrated reasoner score very differently here — by design.

---

## TL;DR

- **3,000 problems** across 5 categories × 5 difficulty levels, generated by 21 parametric templates with **closed-form solutions** (correctness by construction; reseedable).
- **6-metric harness** that scores point accuracy, interval coverage, ECE, overconfidence, framework match, and per-(category, difficulty) breakdowns; bootstrap 95% CIs and paired McNemar tests on every comparison.
- **Validated test split** of 301 problems with statistical power to discriminate tier-scale capability gaps as small as **11.6 pp** ($p < 10^{-3}$).
- **First leaderboard** on five hosted open-weights models — 51-pp spread in Accuracy@CI; Spearman $\rho = -1.00$ between accuracy and ECE; safety-relevant overconfidence rate collapses 10× across the panel.
- **One-line repro**: `python evaluation/run_eval.py --model <name> --save && python evaluation/aggregate_real_llm.py`.

---

## Headline finding

On the open-weights panel we measured, **miscalibration on MURU-BENCH is dominated by capability, not by an independent metacognitive deficit**. Every gain in Accuracy@CI comes bundled with a monotone reduction in Expected Calibration Error (Spearman $\rho = -1.00$, Pearson $r = -0.99$). The simulator validation in §5 of the paper deliberately admits accuracy/calibration dissociation; real models on this panel do not exhibit it.

A second, more actionable finding: **single-number leaderboards mask category-shaped capability holes**. Two otherwise-strong models drop to ≤ 19% on Decision-Under-Uncertainty problems despite scoring 90–100% on the other four categories — invisible to any aggregate metric.

## Quick start

### Install

```bash
git clone https://github.com/swetank18/MURU_proj.git
cd MURU_proj
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### Inspect a problem

```bash
python scripts/sample.py                                                  # 3 random
python scripts/sample.py --category adversarial_ambiguity --difficulty 5  # filtered
python scripts/sample.py --id MURU-0001                                   # specific
```

### Validate the dataset

```bash
make validate
```

### Run simulated baselines (no API key)

```bash
python evaluation/run_baselines.py --save
python evaluation/analyze_results.py
```

### Evaluate a real model

```bash
# Free tier — Groq hosts Llama-3.x, Llama-4-Scout, GPT-OSS, Qwen3, etc.
export GROQ_API_KEY=...
python evaluation/run_eval.py --model llama-3.3-70b --save
python evaluation/run_eval.py --model gpt-oss-120b --save

# OpenRouter — unified gateway for many open models (Nemotron, GLM, Hermes-405B, ...)
export OPENROUTER_API_KEY=...
python evaluation/run_eval.py --model "or:nvidia/nemotron-3-super-120b-a12b:free" --save

# Frontier (paid)
export OPENAI_API_KEY=...        # or ANTHROPIC_API_KEY / GOOGLE_API_KEY
python evaluation/run_eval.py --model gpt-4o --save

# Aggregate to paper-ready bootstrap CIs and LaTeX:
python evaluation/aggregate_real_llm.py
python evaluation/leaderboard.py --update-readme
```

### Tests + CI

```bash
make test
```

GitHub Actions runs the same suite on Python 3.10 / 3.11 / 3.12 on every push and PR.

## Reproduce the paper

```bash
# Regenerate the dataset (deterministic given the seed)
python scripts/generate_problems.py --all --n 3000 --seed 2026 --validate
python scripts/split_data.py --source data/train/ --seed 42

# Bootstrap CIs and McNemar tests (Section 5)
python evaluation/bootstrap_analysis.py

# Figures
python scripts/generate_figures.py

# Compile the paper (one-time tectonic install)
sh .tectonic_install.sh
make paper
```

Every leaderboard row reconstructs bit-exactly from a single per-model JSON archive in `evaluation/baselines/` (raw responses, parsed predictions, computed metrics).
